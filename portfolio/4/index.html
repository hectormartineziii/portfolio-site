Why a good README is crucial to open source projects

This week I met with one of my mentors, Ivan. He was going through some of my github projects when he said something to the extent of: You have a lot of good code here, but no one will notice it if you don't have a good README.

For a while I was of the mind "Code is self-documenting". And to an extent, that is true if someone knows what your code is supposed to do in the first place. The README helps out here a bunch.

Okay, I've learned my lesson. For the next week I'm going to start making better READMEs to my open github projects.
